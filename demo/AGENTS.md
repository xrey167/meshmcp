# demo

## Purpose
Run scripts for the live mesh demo: one gateway, several MCP backends, and demo agent apps, watched together in the Control Room. The scripts wire up the pieces described in `docs/DEMO.md`.

## Key Files
| File | Description |
|------|-------------|
| `run-mesh.ps1` | Windows PowerShell launcher for the demo mesh. |
| `run-mesh.sh` | POSIX shell launcher for the demo mesh. |
| `trace.jsonl` | Sample gateway-wide trace output (artifact; regenerated by runs). |

## For AI Agents

### Working In This Directory
- Two parallel launchers (PowerShell + sh) — keep them behaviorally equivalent when editing.
- They build `mcpserver.exe` and run `meshmcp serve` + `room`/`dash` + `agent` roles; they expect a valid `NB_SETUP_KEY` in the environment.
- `trace.jsonl` is a generated artifact, not source — don't hand-edit it.

## Dependencies

### Internal
- The root `meshmcp` binary, `cmd/mcpserver`, and `examples/*.yaml`.
