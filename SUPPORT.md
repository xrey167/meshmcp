# Support

## Something's broken

1. `meshmcp doctor` — pre-flight checks on your config.
2. `meshmcp diag --bundle diag.tar.gz` — collects the redacted config, doctor
   report, audit chain verdict, and versions into one shareable file.
3. Open a [bug report](.github/ISSUE_TEMPLATE/bug_report.md) with the bundle
   attached.

Operational emergencies (stolen device, suspected audit tamper, key rotation)
are covered step-by-step in [docs/RUNBOOK.md](docs/RUNBOOK.md).

## Security vulnerabilities

Never in a public issue — see [SECURITY.md](SECURITY.md) for the private
reporting channel.

## Questions

Open a GitHub issue with the `question` label. There is no chat channel or SLA
during the pre-1.0 phase; issues are the one place answers accumulate where
the next person can find them.
